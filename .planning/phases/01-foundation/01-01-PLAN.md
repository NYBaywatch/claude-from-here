**src/dllmain.cpp** -- DLL entry point + class factory:

```cpp
// Key elements:
// 1. Global g_hModule (HMODULE) captured in DLL_PROCESS_ATTACH
// 2. Global g_cDllRef (long) for DLL reference counting
// 3. DllMain: store hModule, no heavy work
// 4. CClassFactory implementing IClassFactory:
//    - CreateInstance: creates CClaudeFromHere, calls QueryInterface
//    - LockServer: InterlockedIncrement/Decrement on g_cDllRef
// 5. DllGetClassObject: checks rclsid == CLSID_ClaudeFromHere, creates CClassFactory
// 6. DllCanUnloadNow: returns S_OK if g_cDllRef == 0, S_FALSE otherwise
// CLSID_ClaudeFromHere defined as: {same-guid-from-manifest}
```

The CLSID constant must be defined with `DEFINE_GUID` or `__declspec(selectany) const CLSID CLSID_ClaudeFromHere = {...}` matching the GUID in AppxManifest.xml exactly.

**src/ClaudeFromHere.cpp** -- Full IExplorerCommand + IObjectWithSite implementation:

Implement class `CClaudeFromHere` inheriting `IExplorerCommand` and `IObjectWithSite` per the research Pattern 1 skeleton. Include these headers: `<windows.h>`, `<shlobj.h>`, `<shlwapi.h>`, `<shobjidl_core.h>`, `<exdisp.h>`.

IUnknown methods:
- `QueryInterface`: support `IID_IUnknown`, `IID_IExplorerCommand`, `IID_IObjectWithSite`
- `AddRef` / `Release`: InterlockedIncrement/Decrement with `delete this` + `InterlockedDecrement(&g_cDllRef)` on Release to zero

IExplorerCommand methods:
- `GetTitle`: return `L"Claude from here"` via `SHStrDupW` (MENU-01, MENU-02)
- `GetIcon`: derive icon path from `GetModuleFileNameW(g_hModule, ...)` then `PathRemoveFileSpecW` + `PathAppendW(L"claude.ico")`, return via `SHStrDupW` (MENU-03, per D-03)
- `GetToolTip`: return `L"Open Claude Code in this directory"` via `SHStrDupW`
- `GetCanonicalName`: set GUID to CLSID_ClaudeFromHere, return S_OK
- `GetState`: return `ECS_ENABLED`
- `GetFlags`: return `ECF_DEFAULT`
- `EnumSubCommands`: return `E_NOTIMPL` (no submenus)
- `Invoke`: The core method. Two code paths per research Pattern 2:
  - **Path A (folder right-click):** If `psiItemArray` is not NULL, call `psiItemArray->GetItemAt(0, ...)` then `psi->GetDisplayName(SIGDN_DESKTOPABSOLUTEPARSING, &pszPath)`
  - **Path B (background right-click):** If Path A fails or psiItemArray is NULL, call `_GetFolderPathFromSite(&pszPath)` using the IObjectWithSite chain: `IServiceProvider -> SID_STopLevelBrowser -> IShellBrowser -> QueryActiveShellView -> IFolderView -> GetFolder -> GetDisplayName`
  - **Launch logic (per D-01, D-02):** Use `SearchPathW(NULL, L"wt.exe", NULL, ...)` to find wt.exe on PATH. If not found, `MessageBoxW(NULL, L"wt.exe not found in PATH. Please install Windows Terminal.", L"Claude from here", MB_OK | MB_ICONERROR)` and return. Same for `claude.exe`. If both found, use `CreateProcessW` to run: `wt.exe -d "<pszPath>" cmd /k claude`. Build the command line as: `L"wt.exe -d \"<pszPath>\" cmd /k claude"`. Use `CREATE_NO_WINDOW` or `0` for dwCreationFlags. Close process and thread handles after CreateProcess.

IObjectWithSite methods:
- `SetSite`: store `punkSite` (AddRef new, Release old)
- `GetSite`: QueryInterface on stored site

Private method `_GetFolderPathFromSite(PWSTR* ppszPath)`:
Implement the full IObjectWithSite traversal chain exactly as shown in research Pattern 2. Each interface must be properly Released after use.

**Critical details:**
- All COM pointers must be properly Released (no leaks)
- `g_hModule` and `g_cDllRef` are `extern` from dllmain.cpp
- `CLSID_ClaudeFromHere` is `extern` from dllmain.cpp
- Constructor: InterlockedIncrement(&g_cDllRef); Destructor: InterlockedDecrement(&g_cDllRef)
