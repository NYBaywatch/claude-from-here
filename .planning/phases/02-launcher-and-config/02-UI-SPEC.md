# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Launcher and Config.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> This phase has two UI surfaces:
> 1. **Config app window** — WinForms (.NET 4.8) GUI for configuring Claude CLI flags
> 2. **Error dialogs** — MessageBoxW dialogs surfaced by the DLL at invoke time

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | WinForms (.NET Framework 4.8) — system-provided controls |
| Icon library | Win32 system icons (MB_ICONERROR, MB_ICONWARNING) for dialogs; config app uses no custom icons beyond window title bar |
| Font | Segoe UI 9pt (WinForms default on Windows 11) |

**Rationale (from CONTEXT.md D-01):** .NET Framework 4.8 ships with Windows 11 — no additional runtime required. WinForms was explicitly chosen for small footprint (~50KB .exe) and zero deployment friction.

**shadcn gate:** Not applicable. This is a Win32/WinForms native app. No web component registry involved.

## Spacing Scale

WinForms uses dialog units (DLU) internally, but pixel equivalents at 96 DPI / Segoe UI 9pt:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between label and its control |
| sm | 8px | Padding within a GroupBox; gap between stacked controls |
| md | 16px | Form edge padding (left, right, top) |
| lg | 24px | Vertical gap between GroupBox sections |
| xl | 32px | Bottom margin above the button row |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Button height: 23px (WinForms default) — not overridden
- Window minimum width: 400px; minimum height: 320px
- Apply Settings/Discard Changes buttons: 75px wide x 23px tall (WinForms standard)

## Typography

WinForms inherits system font. No custom typography stack.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / Label | 9pt Segoe UI (~12px) | Regular (400) | System default (~1.3) |
| GroupBox heading | 9pt Segoe UI | Regular (400) | System default |
| Window title | 9pt Segoe UI (title bar, OS-rendered) | Regular (400) | OS-managed |
| Button text | 9pt Segoe UI | Regular (400) | System default |

No custom font sizes. No custom weights. WinForms on Windows 11 renders Segoe UI 9pt by default — use `this.Font = SystemFonts.MessageBoxFont` to inherit correctly.

## Color

WinForms uses system color scheme — no hardcoded hex values. All colors MUST reference `SystemColors` so the app respects the user's Windows light/dark mode and High Contrast settings.

| Role | SystemColors Reference | Usage |
|------|----------------------|-------|
| Dominant (60%) | `SystemColors.Control` | Form background, GroupBox interior |
| Secondary (30%) | `SystemColors.ControlLight` | Not explicitly used — WinForms handles automatically |
| Accent (10%) | `SystemColors.Highlight` | Focused TextBox border, selected DropDown item |
| Destructive | `SystemColors.ControlText` on `MessageBoxW` with `MB_ICONERROR` | Error dialog icon only |

Accent reserved for: focused input border (automatic WinForms behavior), selected dropdown item (automatic). No manual accent color painting required.

No hex values in source code. No hardcoded `Color.FromArgb(...)` calls.

## Surface 1: Config App Window

**Window title:** "Claude From Here — Settings"

**Primary focal point:** Apply Settings button — positioned at bottom-right, standard Windows dialog convention.

### Layout

Single-pane window, no tabs, no scrolling required. All controls visible on open.

```
+------------------------------------------+
| Claude From Here — Settings              |
+------------------------------------------+
|                                          |
|  [Claude CLI Flags]                      |   <- GroupBox
|    Model:       [dropdown v]             |
|    Verbose:     [x] Enable verbose mode  |
|    Allowed Tools: [text field          ] |
|    Extra Flags:   [text field          ] |
|                                          |
|  [Path Detection]                        |   <- GroupBox (read-only status)
|    claude.exe:  Found at C:\...\claude   |
|    wt.exe:      Found at C:\...\wt.exe   |
|                                          |
|    [Discard Changes]  [Apply Settings]   |
+------------------------------------------+
```

### Controls (from CONTEXT.md D-05, D-06)

| Control | Type | Label | Behavior |
|---------|------|-------|----------|
| Model | ComboBox (DropDownList) | "Model:" | Populated with known Claude model IDs; first item is blank (no flag passed) |
| Verbose | CheckBox | "Enable verbose mode" | Checked = `--verbose` appended; unchecked = omitted |
| Allowed Tools | TextBox | "Allowed Tools:" | Raw value passed as `--allowedTools <value>`; empty = omitted |
| Extra Flags | TextBox | "Extra Flags:" | Appended verbatim to command line; empty = omitted |
| Apply Settings | Button | "Apply Settings" | Writes to registry, closes window |
| Discard Changes | Button | "Discard Changes" | Discards unsaved changes, closes window |

### Model Dropdown Values (source: Claude's Discretion)

| Display | Flag value passed |
|---------|------------------|
| (default — no flag) | _(blank — omit --model)_ |
| claude-opus-4-5 | `claude-opus-4-5` |
| claude-sonnet-4-5 | `claude-sonnet-4-5` |
| claude-haiku-3-5 | `claude-haiku-3-5` |

### Path Detection GroupBox

- Read-only display — no user input
- Runs detection on form load using same logic as DLL (SearchPathW first, then App Paths registry)
- Each row shows: label + detected path, or label + "Not found" in red (`Color.Red`) if missing
- "Not found" is the only case where a non-system color is used (explicit exception for error signal)

### On Apply Settings Behavior

1. Validate: warn (not block) if Extra Flags contains shell-unsafe characters that could break `CreateProcessW`. Non-blocking `MessageBox.Show` with MB_ICONWARNING.
2. Write all values to `HKCU\Software\ClaudeFromHere` (from CONTEXT.md D-03, D-04).
3. Close window with `DialogResult.OK`.

### On Discard Changes Behavior

Close window immediately. No confirmation dialog.

## Surface 2: Error Dialogs (MessageBoxW)

Surfaced by the DLL in `Invoke()` when a required executable is not found. Decision D-11/D-12 locked this as `MessageBoxW` — no custom dialog windows.

### claude.exe Not Found

| Field | Value |
|-------|-------|
| Title | "Claude From Here" |
| Icon | MB_ICONERROR |
| Buttons | MB_OK |
| Body copy | "Claude Code was not found on this machine.\n\nTo install, run in any terminal:\n    npm i -g @anthropic-ai/claude-code\n\nAfter installing, restart Windows Explorer (or sign out and back in)." |

### wt.exe Not Found

| Field | Value |
|-------|-------|
| Title | "Claude From Here" |
| Icon | MB_ICONERROR |
| Buttons | MB_OK |
| Body copy | "Windows Terminal was not found on this machine.\n\nTo install, open Microsoft Store and search for 'Windows Terminal', or run:\n    winget install Microsoft.WindowsTerminal\n\nAfter installing, restart Windows Explorer (or sign out and back in)." |

## Interaction States

### Config App

| Control | States |
|---------|--------|
| Model ComboBox | Normal, focused (system highlight), dropdown open |
| Verbose CheckBox | Unchecked (default), checked |
| AllowedTools TextBox | Empty (default), filled, focused |
| Extra Flags TextBox | Empty (default), filled, focused |
| Apply Settings Button | Normal, focused, pressed (all system-rendered) |
| Discard Changes Button | Normal, focused, pressed (all system-rendered) |
| Path status row | "Found at {path}" (black text), "Not found" (red text) |

### Error Dialogs

| State | Trigger |
|-------|---------|
| claude.exe not found | `SearchPathW` + App Paths registry both return no result for `claude.exe` |
| wt.exe not found | `SearchPathW` + App Paths registry + execution alias both return no result for `wt.exe` |
