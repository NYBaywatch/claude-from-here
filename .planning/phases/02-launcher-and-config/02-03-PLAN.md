**Modify `scripts/register.ps1`** to add a config app build step at the end of Step 0 (after the CMake build and DLL copy, before Step 1 certificate setup).

Add the following block after the existing DLL copy logic (after the line `Write-Host "    Stub EXE ready at: $BuildDir\ClaudeFromHere.exe"`) and before the `} else {` branch for `-SkipBuild`:

```powershell
    # Build config app (WinForms .NET 4.8)
    Write-Host "    Building config app (ClaudeFromHereConfig)..."
    $ConfigCsproj = "$PSScriptRoot\..\src\ClaudeFromHereConfig\ClaudeFromHereConfig.csproj"
    if (Test-Path $ConfigCsproj) {
        Invoke-ExternalTool "dotnet build config app" "dotnet" @(
            "build", $ConfigCsproj,
            "--configuration", "Release",
            "--nologo",
            "-v", "minimal"
        )
        if (Test-Path "$BuildDir\ClaudeFromHereConfig.exe") {
            Write-Host "    Config app ready at: $BuildDir\ClaudeFromHereConfig.exe"
        } else {
            Write-Warning "    Config app build succeeded but exe not found at $BuildDir\ClaudeFromHereConfig.exe"
        }
    } else {
        Write-Warning "    Config app .csproj not found at $ConfigCsproj -- skipping config app build"
    }
```

Also add the same config app build in the `-SkipBuild` else branch (after the existing DLL check), so that `-SkipBuild` still builds the config app (it's a separate dotnet project, not affected by CMake):

```powershell
    # Still build config app even with -SkipBuild (it's a dotnet project, not CMake)
    $ConfigCsproj = "$PSScriptRoot\..\src\ClaudeFromHereConfig\ClaudeFromHereConfig.csproj"
    if (Test-Path $ConfigCsproj) {
        Write-Host "    Building config app..."
        Invoke-ExternalTool "dotnet build config app" "dotnet" @(
            "build", $ConfigCsproj,
            "--configuration", "Release",
            "--nologo",
            "-v", "minimal"
        )
    }
```

Do NOT change any other steps in register.ps1. Steps 1-7 remain exactly as they are.
