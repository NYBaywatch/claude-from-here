  <action>
**Step A: Resolve the Azure Trusted Signing certificate Subject**

Run the following to discover the certificate profile's Subject:
```powershell
# Option 1: Azure CLI query (if az trustedsigning extension available)
az trustedsigning certificate-profile list --account-name <account> --resource-group <rg> 2>$null

# Option 2: Check the scanner project's AppxManifest or signing config for the CN pattern
# The same Azure Trusted Signing account may use the same or similar CN
```

If the Azure CLI command is not available or the user needs to provide the CN manually, check if the `sign` tool outputs the certificate Subject during a test signing operation. Alternatively, check the Azure Portal.

**The user may need to provide the exact Publisher CN.** If discoverable via CLI, update `package/AppxManifest.xml` line 16 `Publisher="CN=ClaudeFromHere"` to match the Azure Trusted Signing certificate's Subject. The format is typically `CN=Something, O=Something, L=City, S=State, C=Country` — use the exact string from the certificate profile.

**Step B: Install Inno Setup if missing**

Check if ISCC.exe exists:
```powershell
Test-Path "C:\Program Files (x86)\Inno Setup 6\ISCC.exe"
```

If not found, install via winget:
```powershell
winget install --id JRSoftware.InnoSetup --accept-package-agreements --accept-source-agreements
```

**Step C: Run the build script**

```powershell
.\build-installer.ps1
```

If Azure Trusted Signing env vars are not set, the build will warn and skip signing. For a fully signed build:
```powershell
$env:CFH_SIGNING_ENDPOINT = "https://eus.codesigning.azure.net/"
$env:CFH_SIGNING_ACCOUNT = "<account-name>"
$env:CFH_SIGNING_PROFILE = "<profile-name>"
.\build-installer.ps1
```

Fix any ISCC compilation errors (syntax errors in .iss, missing source files, etc.) by editing `installer/ClaudeFromHere.iss` or `build-installer.ps1` as needed.

**Step D: Verify output**

Confirm `dist/ClaudeFromHere-Setup.exe` exists and report its file size.
