  Flags: `runhidden waitprocuntilterminated`
  RunOnceId: `UnregisterMSIX`

- Restart Explorer:
  Flags: `runhidden waitprocuntilterminated`
  RunOnceId: `RestartExplorer`

**[Code] section** (per Pitfall 5 — upgrade DLL lock prevention, per D-07):
Pascal `CurStepChanged(ssInstall)` callback that runs BEFORE file copy:
1. `Remove-AppxPackage` existing ClaudeFromHere package (idempotent)
2. `Stop-Process -Name explorer -Force` to release DLL lock
3. `Start-Sleep -Seconds 2` to let Explorer shut down fully

Use `Exec('powershell.exe', ...)` with `SW_HIDE` and `ewWaitUntilTerminated`.
Do NOT use double-brace escaping inside [Code] Pascal strings — that escaping is only needed in [Run]/[UninstallRun] Parameters. In Pascal `Exec()` string arguments, use normal single quotes for PowerShell and normal braces.

**Brace escaping rules** (critical — Pitfall 1):
- In `[Run]` and `[UninstallRun]` `Parameters=` values: double all braces (`{` -> `{{`, `}` -> `}}`) EXCEPT Inno Setup constants like `{app}` which stay single-braced
- In `[Code]` Pascal strings: NO brace escaping needed — Pascal strings are not preprocessed by the Inno Setup constant expander
- Path quoting for spaces (Pitfall 2): wrap `{app}` expansions in single quotes inside PowerShell `-Command` strings

**Per D-08:** Do NOT add any `[Registry]` section that deletes `HKCU\Software\ClaudeFromHere`. User settings survive uninstall.

Create `build-installer.ps1` in the project root. This script orchestrates the full build pipeline: compile DLL, build config app, copy assets, pack MSIX, sign MSIX with Azure Trusted Signing, and invoke ISCC to produce the final `.exe`.

**Structure (6 steps):**

**Step 1: Build DLL via CMake + MSVC**
Reuse the same pattern from register.ps1 Step 0:
- `$CMake = "C:\Program Files (x86)\Microsoft Visual Studio\2022\BuildTools\Common7\IDE\CommonExtensions\Microsoft\CMake\CMake\bin\cmake.exe"`
- Configure: `cmake -S . -B build -G "Visual Studio 17 2022" -A x64`
- Build: `cmake --build build --config Release`
- Copy `build\Release\ClaudeFromHere.dll` and `build\Release\ClaudeFromHere.exe` to `build\`

**Step 2: Build config app**
- `dotnet build src\ClaudeFromHereConfig\ClaudeFromHereConfig.csproj --configuration Release --nologo -v minimal`
- Verify `build\ClaudeFromHereConfig.exe` exists after build

**Step 3: Copy assets to build directory**
- Copy `assets\claude.ico` to `build\claude.ico`
- Copy `package\AppxManifest.xml` to `build\AppxManifest.xml`
- Copy `package\Assets\*` to `build\Assets\` (create dir if needed)

**Step 4: Pack MSIX**
- `$MakeAppx = "C:\Program Files (x86)\Windows Kits\10\bin\10.0.26100.0\x64\makeappx.exe"`
- `& $MakeAppx pack /o /d package /nv /p build\ClaudeFromHere.msix`
- Check `$LASTEXITCODE -ne 0` and throw on failure

**Step 5: Sign MSIX with Azure Trusted Signing** (per D-01, D-02)
Read env vars with fallback pattern (same as scanner project):
```powershell
$tsEndpoint = if ($env:CFH_SIGNING_ENDPOINT) { $env:CFH_SIGNING_ENDPOINT } else { "https://eus.codesigning.azure.net/" }
$tsAccount  = if ($env:CFH_SIGNING_ACCOUNT)  { $env:CFH_SIGNING_ACCOUNT  } else { $null }
$tsProfile  = if ($env:CFH_SIGNING_PROFILE)  { $env:CFH_SIGNING_PROFILE  } else { $null }
```

If `$tsAccount` or `$tsProfile` is null, print a warning and skip signing (allow unsigned builds for dev). Otherwise:
```powershell
sign code artifact-signing "build\ClaudeFromHere.msix" `
    --artifact-signing-endpoint $tsEndpoint `
    --artifact-signing-account $tsAccount `
    --artifact-signing-certificate-profile $tsProfile `
    --azure-credential-type azure-cli
```
Check `$LASTEXITCODE -ne 0` and throw "MSIX signing failed".

**Step 6: Build installer with ISCC**
- `$ISCC = "C:\Program Files (x86)\Inno Setup 6\ISCC.exe"`
- Check `Test-Path $ISCC` — throw with install instructions if missing
- `& $ISCC "installer\ClaudeFromHere.iss"`
- Check `$LASTEXITCODE -ne 0` and throw "ISCC compilation failed"
- Report output path: `installer\Output\ClaudeFromHere-Setup.exe` (Inno Setup default output dir is `Output` relative to .iss location, but set `OutputDir=..\dist` in the .iss to put it in `dist/` at project root)

**IMPORTANT UPDATE to Task 1:** Add `OutputDir=..\dist` to the [Setup] section of the .iss file so the final exe lands in `dist/ClaudeFromHere-Setup.exe` at project root.

**Parameters:**
- `[switch]$SkipBuild` — skip Steps 1-2 (reuse existing compiled artifacts)
- `[switch]$SkipSign` — skip Step 5 (for local dev builds without Azure signing)

**Error handling:**
- `$ErrorActionPreference = "Stop"` at top
- Each external tool invocation checks exit code
- Use same `Write-Step` and `Invoke-ExternalTool` helper functions as register.ps1

**Script header:** Include `#Requires -Version 5.1` and a synopsis comment block describing usage and env vars.
